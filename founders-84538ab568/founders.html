PRD Gap Analysis · Action Items

What we need to do

The PRD is a strong scaffold but still mostly placeholders. Benchmarked against the CrewCares v2 PRD, here's the prioritized work to make it review-ready. Owners are starting assignments — adjust as needed.

P0 · Critical — do first
  • Write the Child-Safety, Privacy & Consent sectionCOPPA (under-13 data + verifiable parental consent), FERPA if selling to schools, parent/teacher account control, data minimization for minors, and safe leaderboard design (no exposing kids' names, locations, or contact). The single most urgent gap.Kate
  • Write the AI Behavior Spec + "why agentic" + hard safety rulesDefine what the AI does, justify agent vs. rules, then 15–20 input→output examples and refusals (never recommend age-inappropriate content; never message a child without guardian visibility; never expose one child's data to another).Kate
P1 · Core PRD rigor
  • Add a testable hypothesisIntervention → outcome → metric, with leading indicators and an explicit falsification condition (e.g., "if engagement rises but the fall reading benchmark doesn't move, we're wrong").Kate
  • Replace user-type labels with 2–3 developed personase.g., the reluctant 4th-grade reader, the time-starved motivated parent, the summer-program coordinator — each with JTBD, pains, trigger moment, key product moment, and build-for / cut-for.Kate
  • Build an evidence-backed Problem sectionCited learning-loss and market stats, 3–5 parent/teacher/student quotes, current workarounds (paper logs, school programs), and the cost of inaction.Kate + research
  • Add Strategic Fit + competitive tableWhy-now + brand wedge, and a named landscape — Beanstack, Scholastic Summer Reading, Renaissance/Accelerated Reader, library programs, Epic!/Reading Eggs — each with its structural gap and our "why we win."Kate
  • Add a Defensibility / MOAT sectionWhat compounds with use (engagement data, a school/district graph, a reading-level model, parent trust), an honest "what we don't have on day one," and falsifiable watch metrics.Kate
P2 · Completeness & launch readiness
  • Specify reading verification / anti-gamingHow reading is verified (quizzes, timers, comprehension checks, parent confirmation) and how leaderboard manipulation is prevented; add the gaming rate to guardrail metrics.Kate
  • Build the metrics table + guardrailsMetric · type · baseline · target · timeframe, plus guardrails that must not get worse (verification-gaming rate, parent-reported pressure, week-1 churn), with a "how targets were set" rationale.Kate
  • Write an eval-driven roadmapName the riskiest assumption, define the MVP that tests it, set quantified go/no-go gates and an eval harness (esp. for AI recommendation quality), plus phases tied to the summer calendar and a rollback plan.Kate
  • Add Non-Goals, Open Questions, and a Stage checklistA true "we are NOT building" list; open questions with working positions + owners (pricing, school vs. consumer GTM, content sourcing, age bands, incentive design); and a Stage-readiness self-assessment.Kate
Research inputs needed
  • Collect real interview quotes3–5 verbatim parent / teacher / student quotes to replace placeholders in the Problem and Persona sections.Research
  • Source the statsSummer learning-loss effect sizes and market sizing (K-8 readers, school/library summer-program spend) with citations.Research
